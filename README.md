# Nationality_MVVM

Very Simple App with API integration and MVVM Architecture.

## Getting Started

This project is a starting point for a Flutter application.

This Simple App Contains, following Factors:

1. API integration with Dio and Retrofit.
2. Getit for Registering the services.
3. MVVM as a Structure.

MVVM stands for Model-View-ViewModel and is a popular architecture pattern in Flutter development. 
The MVVM pattern separates the app into three distinct parts:
## Model: The data and business logic of the app
## View: The user interface of the app
## ViewModel: The mediator between the model and the view
## Pros:

1. Provides a clear separation of concerns
2. Supports code reusability and maintainability
3. Facilitates unit testing
4. Supports data binding, making it easier to update the view

## Cons:
1. Can become complex to implement for larger apps
2. This can lead to a large number of files in the codebase
3. This can result in redundant code
